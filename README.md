# ⌁ 開發板接線模擬器

一個專為國小科技課程設計的**接線規劃工具**：上傳開發板與感測器的實際照片、標記腳位，就能讓學生用「點擊」的方式模擬杜邦線接線，先在螢幕上規劃、驗證，再動手實作，減少接錯線燒壞模組的風險。

支援 **Arduino、ESP32、micro:bit** 等任何開發板——因為底圖與感測器圖片都由老師自行上傳，不受廠牌限制。

> 單一 HTML 檔案、無需安裝、無需後端伺服器、可離線使用，電腦與平板觸控皆可操作。![操作畫面](screenshot.png)

## ✨ 特色功能

| 功能 | 說明 |
|------|------|
| 📷 自訂圖片 | 上傳開發板照片當固定底圖、感測器照片進圖庫，同款可放多個 |
| 📍 標記腳位 | 在照片上點擊針腳並命名（5V、GND、D9、A0⋯⋯），感測器只需標記一次，所有實例共用 |
| 🔗 點擊接線 | 點起點腳位 → 即時預覽線跟隨游標 → 點終點完成；線會吸附腳位，拖曳感測器或縮放底圖時自動跟隨 |
| 🎨 自動配色 | 電源（V）固定紅色、接地（G）固定黑色、訊號線自動輪流分配不重複顏色 |
| ⚠️ 錯誤判斷 | 電源與接地互接（短路）時，線變紅色虛線並顯示警示；「檢查接線」一鍵列出所有錯誤 |
| 🔦 聚焦模式 | 淡化開發板上未標記腳位的區域，只保留使用中的腳位清晰，視線一眼對焦 |
| 🔄 旋轉縮放 | 感測器可縮放（30%–250%）、旋轉（滑桿微調或一鍵轉 90°），腳位與接線同步跟隨 |
| ✏️ 手繪註記 | 灰色畫筆自由手繪圈重點；橡皮擦劃過即可刪除手繪線或接線 |
| 💾 儲存進度 | 匯出／匯入 `.json` 專案檔，學生可分次完成或繳交原始檔 |
| 🌐 匯出成品網頁 | 將圖片、腳位、版面**全部內嵌**成一個新的 HTML 檔，直接發布給學生，開啟即用 |
| 🖼️ 匯出圖片 | 一鍵下載高解析度 PNG 接線圖，可貼進學習單或上傳作業平台 |

## 🚀 快速開始

### 方法一：直接開啟

下載 `index.html`，用瀏覽器（建議 Chrome / Edge / Safari）打開即可，**不需要網路**（僅介面字型需連線，離線時自動改用系統字型）。

### 方法二：發布到 GitHub Pages（推薦）

1. 建立 GitHub 儲存庫，將模擬器檔案改名為 `index.html` 上傳到根目錄。
2. 進入儲存庫 **Settings → Pages**，Source 選擇 **Deploy from a branch**，Branch 選 `main`、資料夾選 `/ (root)`，按 Save。
3. 等待 Actions 中的「pages build and deployment」流程跑完（deploy 出現綠色勾勾）。
4. 網址即為 `https://<你的帳號>.github.io/<儲存庫名稱>/`，可直接分享給學生。

> 💡 之後每次更新：上傳新檔覆蓋 `index.html` → 等 Actions 綠燈 → 學生端按 `Ctrl + F5` 強制重新整理即可看到新版。

## 👩‍🏫 教師備課流程

1. **上傳開發板圖片**（右欄「圖片」）：照片會置中成為固定底圖，可用「底圖大小」滑桿調整。建議使用光線均勻、垂直俯拍的照片，解析度長邊 1200px 以上，腳位絲印文字才清晰。
2. **上傳感測器圖片**：可一次選多張並命名。建議先去背存成 PNG，畫面更乾淨。
3. **標記腳位**（左欄切到「標記腳位」模式）：
   - 點開發板照片上的針腳位置 → 輸入名稱（如 `5V`、`GND`、`D9`）。
   - 把感測器加入畫布，點它的針腳 → 輸入名稱（如 `V`、`G`、`S`）。**同款感測器只需標一次。**
   - 命名規則：名稱含 `V / VCC / VIN / 5V / 3V3` 自動歸類為**電源（紅點）**；含 `G / GND` 為**接地（黑點）**；其餘為**訊號（黃點）**。
   - 點已存在的腳位可重新命名；名稱清空並確定即刪除。換課程配置時可按「清除開發板腳位」一鍵清空（感測器腳位保留）。
4. **匯出成品網頁**（右欄「檔案」）：產生內嵌所有圖片與腳位的新 HTML 檔，將它改名 `index.html` 上傳 GitHub 即完成發布——學生打開就有備好的教材，不需重新上傳圖片。

## 🧑‍🎓 學生使用流程

1. 點「感測器圖庫」縮圖，把需要的模組加入畫布；用「移動」模式拖曳到定位，必要時縮放、旋轉。
2. 切到「接線」模式：點起點腳位 → 點終點腳位，完成一條線。點空白處或按 `Esc` 可取消。
3. 接錯了？用「橡皮擦」劃過該線，或按「復原」。
4. 按「檢查接線」確認沒有電源／接地互接的錯誤。
5. 按「匯出圖片」下載 PNG 繳交，或「儲存進度」保留 `.json` 下次繼續。

## 🖥️ 介面配置（三欄式）

- **左欄（操作）**：模式切換、配色規則圖例、畫布動作（檢查／復原／清除／匯出圖片）。
- **中欄**：感測器圖庫＋大畫布，畫布使用整個瀏覽器寬度（內部解析度 1600×800）。
- **右欄（設定）**：圖片上傳與顯示選項（顯示名稱、聚焦模式）、選取感測器的屬性（大小、旋轉、刪除）、檔案功能。
- 寬度 1000px 以下（平板直式、手機）自動改為單欄，工具區排在畫布上方。

|------|------|

| `Delete` / `Backspace` | 刪除選取的感測器 |

## 📁 檔案格式說明

- **`index.html`**：模擬器本體（或已內嵌教材的成品網頁），單一檔案包含全部程式與樣式。
- **`接線模擬器專案.json`**：儲存進度產生的專案檔，內含底圖、感測器圖片（base64）、腳位、接線與手繪內容，可用「讀取進度」載回。
- **`開發板接線模擬器_成品.html`**：「匯出成品網頁」產生的發布版，已內嵌教材，開啟即用；本身仍具備完整功能，也能再次匯出新的成品。
- **`接線圖.png`**：匯出圖片產生的高解析度成果圖（3200×1600）。

舊版專案檔可直接被新版讀取（腳位座標會自動轉換為相對於底圖的格式）。

## 🔧 技術說明

- 純前端：HTML + CSS + Vanilla JavaScript + Canvas 2D，無任何框架與相依套件。
- 無後端、無資料庫、不蒐集任何資料；所有內容僅存在使用者下載的檔案中。
- 圖片上傳後自動壓縮（底圖長邊 1600px、感測器 900px），控制成品網頁檔案大小。
- 相容 Chrome、Edge、Safari、Firefox 之近年版本；觸控操作採 Pointer Events。

## 🏫 教學應用建議

- **模擬 → 檢核 → 實作三部曲**：學生先在模擬器完成接線並匯出 PNG，通過教師或同儕檢核後才領實體材料。
- **偵錯偵探遊戲**：教師故意接錯幾條線後匯出成品網頁，讓學生分組找碴並用「檢查接線」驗證。
- **任務卡關卡制**：每關一個專案檔（Lv1 點亮 LED → Lv2 超音波測距 → Lv3 倒車雷達），通關集點。
- **跨板教材庫**：Arduino、ESP32、micro:bit 各做一個成品網頁，同一套感測器照片重複使用。

## ❓ 常見問題

**Q：發布到 GitHub Pages 後網頁沒更新？**
A：先到儲存庫 Actions 確認「pages build and deployment」已跑完綠燈；再按 `Ctrl + F5` 強制重新整理（Pages 與瀏覽器都有快取）。若 deploy 長時間卡在 Queued，多半是 GitHub 端壅塞，可取消後 Re-run，或查看 [githubstatus.com](https://www.githubstatus.com)。

**Q：成品網頁檔案會不會太大？**
A：圖片已自動壓縮，一般 1 張底圖＋5～8 顆感測器約 1–3 MB，GitHub 單檔上限 25 MB（網頁上傳）綽綽有餘。

**Q：換了一張新的開發板照片，腳位還在嗎？**
A：腳位以「相對於底圖」的比例儲存，更換構圖不同的照片後位置會偏移，建議按「清除開發板腳位」重新標記。

**Q：平板可以用嗎？**
A：可以，全功能支援觸控。標記腳位需要較精準的點擊，建議搭配觸控筆。
